# 世界漫游指南 - 架构文档

## 项目概述

**项目名称**: 世界漫游指南 (World Travel Guide) v1.1.0

**项目定位**: AI 驱动的互动小说引擎 (AI-Powered Interactive Fiction Engine)

**技术栈**:
- 前端框架: React 19 + TypeScript
- 构建工具: Bun (原生打包，Bun.build())
- 状态管理: Zustand + React Context (useReducer)
- 持久化: IndexedDB (存档) + localStorage (配置)
- AI API: OpenAI-compatible SSE 流式接口
- 样式: 纯 CSS + CSS 变量设计系统 + CSS Layers (base/layout/state/theme/print/a11y)

### 管线执行流程

![管线架构](diagrams/pipeline.svg)

1. **PromptAssemble** — 从 PresetPack 构建系统提示
2. **ContextPrepare** — 清理消息上下文，应用正则脚本
3. **VariableInject** — 注入当前变量状态到提示词
4. **WorldBookScan** — 扫描世界书条目并注入
5. **ModuleApply** — 应用世界模块（生存、经营、骰子等）
6. **APIRequest** — 发送 SSE 流式请求
7. **ResponseExtract** — 从响应中提取结构化数据
8. **VariableUpdate** — 更新变量状态

## 路由与导航

本项目**不使用 React Router**，导航完全由状态驱动。

### 顶层路由

`GameContext` 内部通过 `useReducer` 管理一个三屏状态机：

```
Screen: 'start' | 'settings' | 'game'
```

### Provider 嵌套层级

```
ErrorBoundary
  └── UISettingsProvider (主题/字体)
        └── GameProvider (导航状态)
              └── AppContent (switch 分发三个 Screen)
```

## 依赖规则

- 上层可以依赖下层，下层不能依赖上层
- 模块之间不能相互引用
- 上层通过接口调用下层
